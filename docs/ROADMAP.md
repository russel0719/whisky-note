# Whisky Note 고도화 로드맵

2026-07-17 기준. v1(시음/구매 기록, 보틀 관리, 아로마 태그, PWA)이 배포된 상태에서
"기록 도구"를 넘어 "취향 아카이브"로 끌어올리기 위한 계획.

## A. 기능 고도화

### Phase 1 — 기록 경험 완성 (v1의 빈 곳 메우기)

| 항목 | 내용 | 비고 |
|---|---|---|
| 수정 기능 | 위스키/시음 노트/보틀 **편집** 화면. 현재 삭제만 가능 | 스키마 변경 없음. 최우선 |
| 사진 업로드 | 보틀·시음 사진 (Supabase Storage, `whisky-note` 버킷) | `whiskies.image_url` 컬럼 기존재. Storage RLS 정책 필요 |
| 위스키 컬러 기록 | 시음 시 위스키 색상(페일 골드~마호가니) 스와치 선택 | `tastings.color` 컬럼 추가 (마이그레이션). 디자인 시그니처와 결합 |
| 검색·필터 고도화 | 노트 전문 검색, 점수 범위·아로마 태그·재구매 의사 필터, 정렬 | 기존 쿼리 확장 |
| 빠른 재기록 | 위스키 상세/대시보드에서 "이 위스키 다시 마심" 원탭 → 시음 폼 프리필 | 바 사용 시나리오 |

### Phase 2 — 인사이트 (쌓인 데이터가 보상이 되는 단계)

| 항목 | 내용 | 비고 |
|---|---|---|
| 취향 프로필 | 아로마 9그룹 **레이더 차트**, 최다 태그, 카테고리·지역별 평균 점수 | `/stats` 페이지. SVG 직접 구현 (외부 차트 라이브러리 지양) |
| 개봉 경과 × 점수 추이 | 같은 보틀의 시음 점수를 개봉 경과일 축으로 그린 라인 — **산화가 맛에 미치는 영향 추적** | bottle_id 연결 + open_date가 이미 있어 데이터 자동 축적. 이 앱만의 차별 기능 |
| 지출 대시보드 | 월별 구매/잔값 지출, 보유 캐비닛 가치(잔량 반영), 원/점수 가성비 | 기존 컬럼으로 계산 가능 |
| 점수 스파크라인 | 위스키 상세에 시음 점수 추이 미니 차트 | |
| 마셔야 할 보틀 | 개봉 후 N개월 경과 + 잔량 남은 보틀을 대시보드에서 리마인드 | 푸시 알림 아님, 배지/섹션으로 |

### Phase 3 — 확장

| 항목 | 내용 | 비고 |
|---|---|---|
| 위시리스트 | 마시고 싶은 위스키. 바 시음 + 재구매 "예" → 자동 제안 | `wishlist` 테이블 or `whiskies.status` |
| 커스텀 아로마 태그 | 개인 태그 추가 | `aroma_tags.user_id uuid NULL` 추가, RLS 정책 확장 |
| 노트 공유 카드 | 시음 노트를 이미지 카드로 렌더 → SNS 공유 | `@vercel/og` 또는 satori |
| 데이터 내보내기 | CSV/JSON export (백업 겸용) | 서버 액션으로 충분 |
| 오프라인 작성 | 바 지하 등 무신호 환경에서 IndexedDB 큐 → 온라인 시 동기화 | Serwist Background Sync. 복잡도 높음, 마지막 |
| 플라이트(비교 시음) | 한 세션에 여러 위스키 나란히 기록·비교 | 스키마 확장 필요, 수요 확인 후 |

## B. 디자인 고도화 — "장인적 다크 럭셔리"

방향: 현재 다크+앰버 토큰 위에 **세리프 디스플레이, 시그니처 데이터 비주얼, 절제된 빛**을
얹어 "고급 위스키 바의 메뉴판" 감각으로. 모든 편차는 `~/.claude/docs/projects/whisky-note.md`에 기록.

### B1. 타이포그래피 — 럭셔리 문법의 핵심

- **세리프 디스플레이 도입**: 위스키 이름·페이지 헤드라인·큰 점수에 명조 계열
  (한글 **Noto Serif KR** + 라틴/숫자 **Playfair Display** 조합, weight 500–600).
  본문/UI는 Pretendard 유지 → serif display + sans body의 이중 구조.
- **Eyebrow 시스템화**: 현재 페이지마다 흩어진 tracking-[0.25em] uppercase 라벨을
  공용 컴포넌트로 통일 (크기/자간/컬러 고정).
- **점수 타이포**: 큰 점수는 세리프 + 가벼운 weight + tabular-nums로 "빈티지 계기판" 느낌.

### B2. 컬러·서피스 정교화

- **점수 밴드 컬러**: 90+ 골드(#e3b563), 80–89 앰버(#c9963f), 70–79 뉴트럴 크림, 그 이하 muted —
  점수가 보이는 모든 곳에 일관 적용.
- **위스키 컬러 스와치 팔레트**: SMWS 계열 색상환 참고한 12단계 (페일 스트로 → 다크 마호가니).
  Phase 1의 color 컬럼과 세트.
- **서피스 한 단계 추가**: 모달/호버용 elevated tile (`#262119` 부근).
- **은은한 빛**: 히어로/로그인에 한해 radial glow(앰버 2–4% 알파) 허용 — 위스키 잔에 떨어지는
  조명의 은유. 남용 금지(페이지당 1곳). design.md "no gradients" 원칙의 명시적 편차로 기록.

### B3. 시그니처 컴포넌트 (이 앱을 기억하게 만드는 것)

1. **점수 다이얼**: 총점을 원형 게이지(SVG, 앰버 아크 + 세리프 숫자)로 — 노트 상세/카드의 얼굴.
2. **아로마 레이더**: 9그룹 레이더 차트, hairline 그리드 + 앰버 면(10% 알파). 취향 프로필의 얼굴.
3. **보틀 실루엣 잔량 게이지**: 현재 막대 게이지를 보틀 모양 SVG 채움으로 교체.
4. **위스키 컬러 스와치**: 원형 스와치 + 라벨. 노트 상세 헤더에 표시.
5. **빈 상태 일러스트**: 글렌캐런 라인아트(1px hairline 스트로크, 앰버 포인트)로 통일.

### B4. 레이아웃·모션

- **대시보드 에디토리얼화**: 최근 최고점/최근 시음 1건을 풀블리드 히어로 타일로
  (세리프 대형 타이포 + 점수 다이얼), 그 아래 유틸리티 그리드. design.md 타일 리듬 문법 적용.
- **데스크톱 2컬럼**: max-w-3xl 단일 컬럼 → 대시보드·상세는 max-w-5xl 2컬럼 그리드.
- **모션 원칙**: 페이지 진입 fade+8px rise(200ms), 카드 호버 hairline→amber 30% 전환,
  다이얼/게이지 값 애니메이션(600ms ease-out). `prefers-reduced-motion` 존중.
  스프링/바운스 금지 — 절제가 럭셔리.
- **스켈레톤 로딩**: 각 라우트 `loading.tsx` — 실제 레이아웃 골격과 동일한 형태.

### B5. 마감

- 커스텀 range 슬라이더 (앰버 트랙/썸, 웹킷·모즈 스타일링).
- 로그인 페이지: radial glow + 보틀 라인아트로 첫인상 강화.
- OG 이미지(기본 1장) — 링크 공유 시 브랜드 노출.
- 대비(WCAG AA)·포커스 링 전수 점검, iOS standalone 세이프에어리어 재점검.

## 추가 구현됨 (2026-07-19)

- **가격 × 만족도 산점도** (/stats): 보틀 시음은 잔당 30ml 환산가, 바 시음은 잔 가격으로 정규화해 한 축에서 비교. 보틀=채운 점, 바=테두리 점.
- **아로마 취향 검색** (/tastings): 태그 조합(AND) + 최소 점수 필터로 내 기록 역검색.

## 구현 순서 제안

1. **디자인 파운데이션** (B1 타이포 + B2 컬러 밴드 + B4 모션 원칙 + 스켈레톤) — 이후 모든 작업의 토대
2. **Phase 1 기능** (편집 → 사진 → 컬러 스와치(B3-4와 세트) → 검색/필터 → 빠른 재기록)
3. **시그니처 비주얼** (B3 다이얼·레이더·보틀 게이지 + B4 대시보드 에디토리얼화)
4. **Phase 2 인사이트** (`/stats`, 개봉 경과 추이, 지출)
5. **Phase 3**은 사용해보며 수요 확인 후 선별

각 단계마다: 마이그레이션이 있으면 타임스탬프 버전으로 추가(전역 규칙 3.2 스텁 주의),
디자인 편차는 프로젝트 문서에 즉시 기록, lint/build 통과 + 실기기 확인 후 push(자동 배포).
